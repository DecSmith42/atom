Properties

SetupBuildInfo

Gets the build target responsible for setting up the build ID, version, and timestamp.

Target SetupBuildInfo { get; }

Property Value

Target

A Target configured to initialize build information variables and report data.

Examples

While SetupBuildInfo is often implicitly part of a default build definition, here's how you might conceptualize its use and the consumption of variables it produces. Assume BuildName is set in the build definition (e.g., by inheriting from IBuildInfo). 

// In your Atom build definition (e.g., Build.cs)
public class MyBuild : BuildDefinition, ISetupBuildInfo
{
    // ISetupBuildInfo members (BuildIdProvider, etc.) would be implicitly available
    // or explicitly implemented if not using a base class that provides them.
    // The BuildName property is inherited from IBuildInfo
    public override string BuildName => "MyAwesomeProject";
    // A custom target that depends on SetupBuildInfo having run
    Target Package => t => t
        .DependsOn(nameof(ISetupBuildInfo.SetupBuildInfo)) // Ensures build info is set up
        .Executes(async () =>
        {
            // Retrieve variables set by SetupBuildInfo
            var buildId = await GetVariable<string>("BuildId");
            var buildVersion = await GetVariable<string>("BuildVersion");
            var buildTimestamp = await GetVariable<string>("BuildTimestamp");
            // Log or use these variables
            LogInformation($"Packaging {BuildName} version {buildVersion} (Build ID: {buildId}, Timestamp: {buildTimestamp})");
            // ... further packaging logic ...
        });
    // Main build flow
    Target Default => t => t.DependsOn(Package);
}
// To run this build (assuming Atom CLI):
// atom Default

Remarks

The SetupBuildInfo target is a foundational step in the Atom build pipeline. It performs several key actions: 

<ul><li>
            Retrieves the build name (from <xref href="DecSm.Atom.Build.BuildInfo.IBuildInfo.BuildName" data-throw-if-not-resolved="false"></xref>) and writes it as a build
            variable named
            "BuildName".
        </li><li>
            Retrieves the build ID (from <xref href="DecSm.Atom.Build.BuildInfo.IBuildInfo.BuildId" data-throw-if-not-resolved="false"></xref>) and writes it as a build variable
            named "BuildId".
        </li><li>
            Retrieves the build version (from <xref href="DecSm.Atom.Build.BuildInfo.IBuildInfo.BuildVersion" data-throw-if-not-resolved="false"></xref>) and writes it as a build
            variable
            named
            "BuildVersion".
        </li><li>
            Retrieves the build timestamp (from <xref href="DecSm.Atom.Build.BuildInfo.IBuildInfo.BuildTimestamp" data-throw-if-not-resolved="false"></xref>), converts it to a string,
            and
            writes it
            as a build variable named "BuildTimestamp".
        </li><li>
            Adds a "Run Information" section to the build report, displaying the build name, version,
            ID, and timestamp.
        </li><li>
            Logs the build ID, version, and timestamp using the configured logger for
            <code>ISetupBuildInfo</code>.
        </li></ul>

This target is marked as hidden (IsHidden()), meaning it's not typically intended for direct invocation by users but is expected to run as a dependency for other targets or as part of an initial setup sequence.

It produces the following build variables, which can be consumed by subsequent targets in the build process: 

<ul><li><span class="term">
            <code>BuildName</code>
        </span>The name of the build.</li><li><span class="term">
            <code>BuildId</code>
        </span>The unique identifier for the build execution.</li><li><span class="term">
            <code>BuildVersion</code>
        </span>The version of the software being built.</li><li><span class="term">
            <code>BuildTimestamp</code>
        </span>The timestamp of the build (as a string).</li></ul>

Successful execution relies on the correct registration and functioning of IBuildIdProvider, IBuildVersionProvider, and IBuildTimestampProvider services within the Atom framework.
