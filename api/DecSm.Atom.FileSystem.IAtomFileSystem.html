Interface IAtomFileSystem

Namespace
DecSm.Atom.FileSystem
Assembly
DecSm.Atom.FileSystem.dll

Defines a specialised file system abstraction for Atom, layering build-aware path resolution on top of the standard System.IO.Abstractions.IFileSystem abstraction.

[PublicAPI]
public interface IAtomFileSystem

Remarks

Path resolution is intentionally separate from file I/O: GetPath(string) queries a prioritized chain of IPathProvider implementations and caches the result. Callers never need to hard-code paths or know which provider resolved a key.

Properties

RootedPath

Remarks

Evaluated eagerly each time the property is read — it is not cached because the working directory can change during the lifetime of the process.

FileSystem

Gets the underlying System.IO.Abstractions.IFileSystem instance used for all file I/O operations.

IFileSystem

Remarks

All System.IO.Abstractions.IFileSystem members on this interface delegate to this property so that consumers of IAtomFileSystem work against a single, consistent abstraction regardless of which concrete file-system backend is registered.

Methods

Returns

Remarks

Prefer this factory method over constructing RootedPath directly so that the resulting path is automatically bound to this IAtomFileSystem rather than to the inner FileSystem.

GetPath(string)

Resolves a well-known path by key, querying the registered IPathProvider chain and caching the result for later calls.

RootedPath GetPath(string key)

Parameters

key string

An application-defined key that identifies the path (e.g. "Root", "Artifacts"). Keys are compared by the individual providers and are conventionally defined as constants.

Returns

RootedPath

A RootedPath corresponding to the key.

Remarks

Results are cached after the first successful resolution so that repeated calls to GetPath("Root") always return the same RootedPath instance without re-querying providers. This also means that if two providers could resolve the same key, only the one with the highest priority (earliest in the ordered list) ever runs.

Exceptions

InvalidOperationException

Thrown when no registered provider can resolve key, or when the resolution depth exceeds the circular-dependency guard threshold.

GetPath<T>()

Resolves the path for a given IPathMarker type by invoking its static Path(IFileSystem) method directly.

RootedPath GetPath<T>() where T : IPathMarker

Returns

RootedPath

A RootedPath for the specified marker type.

Type Parameters

T

A type that implements IPathMarker.

Remarks

Unlike GetPath(string), this overload bypasses the provider chain and the cache entirely — it calls T.Path(this) on every invocation. Use it for paths that are statically determined by the marker type itself, such as source-generated project paths.
