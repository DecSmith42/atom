Properties

NugetConfigPath

Gets the path to the default NuGet configuration file (NuGet.Config) for the current operating system.

RootedPath NugetConfigPath { get; }

Property Value

RootedPath

Remarks

This property determines the standard location where NuGet configuration files are expected. On Windows, it's typically %APPDATA%\NuGet\NuGet.Config. On Linux/macOS, it's typically $HOME/.nuget/NuGet.Config.

NugetDryRun

Gets a value indicating whether NuGet write operations (like pushing packages) should be performed as a dry run.

[ParamDefinition("nuget-dry-run", "Whether to perform a dry run of nuget write operations.", ParamSource.All, null)]
bool NugetDryRun { get; }

Property Value

bool

Remarks

When true, NuGet operations that modify remote feeds will be simulated and logged, but no actual changes will be made. This is useful for testing build configurations.

CreateNugetConfigOverwriteScope(NugetFeed[], bool, CancellationToken)

Creates a TransformFileScope that temporarily overwrites the default NuGet.Config file with a configuration that includes the specified NuGet feeds and their credentials.

[PublicAPI]
Task CreateNugetConfigOverwriteScope(NugetFeed[] feeds, bool skipIfExists = false, CancellationToken cancellationToken = default)

Parameters

feeds NugetFeed[]

An array of NugetFeed objects to include in the configuration.

skipIfExists bool

If true, and all specified feeds are already present in the existing NuGet.Config, no overwrite will occur, and an empty scope will be returned.

cancellationToken CancellationToken

A cancellation token to observe while waiting for the task to complete.

Returns

Task

A System.Threading.Tasks.Task<TResult> that returns a TransformFileScope instance. Disposing this scope will restore the original NuGet.Config file.

Remarks

This method generates a NuGet.Config XML structure including <packageSources> and < packageSourceCredentials> based on the provided feed information. Passwords are included as clear text if specified via PlainTextPassword.

PushProject(string, string, string, RootedPath?, CancellationToken)

Pushes a NuGet package for a specific project to a NuGet feed.

[PublicAPI]
Task PushProject(string projectName, string feed, string apiKey, RootedPath? configFile = null, CancellationToken cancellationToken = default)

Parameters

projectName string

The name of the project whose package is to be pushed.

feed string

The NuGet feed URL to push the package to.

apiKey string

The API key for authenticating with the NuGet feed.

configFile RootedPath

Optional path to a NuGet configuration file to use instead of the default one.

cancellationToken CancellationToken

A cancellation token to observe while waiting for the task to complete.

Returns

Task

A System.Threading.Tasks.Task representing the asynchronous operation.

Remarks

This method expects the project to have been built and its NuGet package (`.nupkg`) to be available in the Atom artifacts directory, named according to the project name and the BuildVersion.

It requires the `dotnet` CLI to be installed and available in the system's PATH.

If NugetDryRun is true, the push operation will be simulated.
