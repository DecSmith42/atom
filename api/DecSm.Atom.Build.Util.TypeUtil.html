Provides flexible type conversion utilities, handling primitives, arrays, and generic collections.

Remarks

This utility class is designed to handle complex type conversions beyond standard .NET capabilities, with special handling for arrays (from comma-separated strings) and generic collections. It supports custom converters for specialized types and is optimized for performance-sensitive scenarios like parameter resolution.

Methods

Convert<T>(string?, Func<string?, T?>?)

Converts a string value to a specified target type, with optional custom conversion logic.

[UnconditionalSuppressMessage("Trimming", "IL2026", Justification = "May fall back to the TypeDescriptor-based Convert(string, Type) helper (marked RequiresUnreferencedCode) when no custom converter, array/generic path, or AOT-safe TryConvert path can handle the input. This is a last-resort conversion; in trimmed/AOT builds callers should prefer passing explicit converters for unsupported types.")]
public static T? Convert<T>(string? stringValue, Func<string?, T?>? converter)

Parameters

stringValue string

The string value to convert. For array types, a comma-separated string is expected.

converter Func<string, T>

An optional custom conversion function that takes precedence over built-in logic.

Returns

T

The converted value, or the default value for the type if conversion is not possible.

Type Parameters

T

The target type to convert to.

Remarks

The method uses a prioritized strategy: custom converter, array conversion, generic collection conversion, and finally, standard System.ComponentModel.TypeDescriptor conversion. It is designed to be AOT-friendly by attempting simple, reflection-free conversions first.
