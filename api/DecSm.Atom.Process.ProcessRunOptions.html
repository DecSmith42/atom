ProcessRunOptions

Remarks

All properties have defaults suited to typical build-automation use: invocation is logged at Microsoft.Extensions.Logging.LogLevel.Information, stdout at Microsoft.Extensions.Logging.LogLevel.Debug, and stderr at Microsoft.Extensions.Logging.LogLevel.Warning. A non-zero exit code throws by default so build steps fail fast without requiring callers to check exit codes manually.

Both stdout and stderr are fully captured into ProcessRunResult regardless of log level, so callers can inspect the full output even when logs are suppressed.

AllowFailedResult

Gets a value indicating whether a non-zero exit code should be tolerated rather than throwing an exception.

public bool AllowFailedResult { get; init; }

Property Value

bool

Remarks

When false (the default), Run(ProcessRunOptions) and RunAsync(ProcessRunOptions, CancellationToken) throw if the exit code is non-zero, which causes the enclosing build target to fail immediately.

Set to true when the caller needs to inspect the result regardless of success — for example, running a linter and deciding separately whether to fail the build. The caller must then check ExitCode explicitly.

TransformOutput

Gets an optional per-line transformation applied to each line of the process's standard output before it is logged and captured.

public Func<string, string?>? TransformOutput { get; init; }

Property Value

Func<string, string>

Remarks

The delegate receives a raw non-null line (null data events are filtered before the delegate is called). Returning null suppresses the line entirely — it is neither logged nor included in Output. This is useful for filtering noisy progress lines or redacting secrets from captured output.

WorkingDirectory

Property Value

Remarks

When null (the default), the process inherits the current application's working directory. Set this when a tool must be run from a specific project or repository root.
