Remarks

Both Output and Error are always populated with whatever the process wrote, regardless of the log levels configured in RunOptions. This means callers can inspect the full output in error-handling logic even when debug logging was disabled during the run.

For processes run with AllowFailedResult set to false (the default), this record is only returned when ExitCode is zero — otherwise IProcessRunner throws before returning.
