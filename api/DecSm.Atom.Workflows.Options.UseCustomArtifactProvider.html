Examples

To enable a custom artifact provider in a workflow:

// In your build definition:
public override IReadOnlyList<WorkflowDefinition> Workflows =>
[
    new("MyWorkflow")
    {
        Options = [UseCustomArtifactProvider.Enabled],
        // ... other workflow configurations
    }
];

Remarks

When Enabled, the workflow will use the StoreArtifact and RetrieveArtifact targets, which delegate to the registered IArtifactProvider implementation.

If this option is disabled, the framework defaults to the native artifact capabilities of the CI/CD platform (e.g., actions/upload-artifact in GitHub Actions).

Enabling this option ensures a consistent artifact management strategy across different CI/CD platforms and local builds.
